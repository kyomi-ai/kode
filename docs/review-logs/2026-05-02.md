### drag-and-drop reordering UI — initial review

**Files**: `kode-leptos/src/wysiwyg/tree_editor.rs`, `kode-leptos/src/wysiwyg.css`, `kode-leptos/Cargo.toml`

| # | Severity | Anti-Pattern Category | File:Line | Description |
|---|----------|----------------------|-----------|-------------|
| 1 | 🟡 | Missing error handling / unhandled event | tree_editor.rs ~703–733 | No `pointercancel` handler. On touch/stylus interruption the browser fires `pointercancel` instead of `pointerup`, leaving `drag_state.active == true` permanently. Body retains `kode-dragging` class, source block retains `kode-block-dragging`, every subsequent `pointermove` runs DOM queries against a phantom drag, and the next stale `pointerup` fires `move_block` with leftover `source_start`/`source_end`/`target_pos`. Fix: add a `pointercancel` closure with the same cleanup path as `pointerup` (reset `active`, remove classes, remove indicator) but without the `move_block` commit, and register/deregister it alongside the other three listeners. |
| 2 | 🟢 | Code quality — unnecessary cast | tree_editor.rs ~541 | `target_el.unchecked_ref::<HtmlElement>().set_pointer_capture(...)` — `target_el` is already `web_sys::Element` and `set_pointer_capture` is defined directly on `Element` in web-sys. The `HtmlElement` cast is misleading. Use `target_el.set_pointer_capture(ev.pointer_id())` directly. |
| 3 | 🟢 | Performance — repeated DOM query per `pointermove` | tree_editor.rs ~569 | `query_selector_all("[data-pos-start]")` is called on every `pointermove` (~60 Hz). The block list is stable during drag (HTML caching is active). Query once in `pointerdown` and store in a `RefCell<Vec<...>>` field on `DragState`. Not blocking for initial launch but will degrade on large documents. |

**Anti-pattern tally**: Missing error handling ×1, Code quality cast ×1, Performance ×1

**Notes**: DragState correctly uses `Cell<T>` (no Leptos signals). HTML caching logic is correct — `version.get()` read unconditionally maintains reactive dependency; `active.set(false)` precedes `notify_up` so the Memo reruns cleanly post-drop. Pointer event closures match the existing `RefCell<Option<(closures...)>>` + `Effect::new` + `.take()` + `on_cleanup` pattern. No `.forget()` on persistent closures. `mount_drag_handles` idempotency guard is correct. Event delegation class check is correct. `move_block` call signature matches `editing.rs:635`. Single-block and zero-block edge cases handled correctly. CSS custom properties follow established pattern. `scrollTop` auto-scroll is valid — container has `overflow-y: auto`.
